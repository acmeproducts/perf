ui-v1 local baseline

Curate device images locally with the same stack-first workflow.

This local-only variant keeps the proven curation flow from ui-v1, but removes cloud sign-in and cloud sync. Metadata is stored in a local manifest/sidecar file when writable access is granted, with IndexedDB used for cached state and handle reuse.

Pick a writable folder for the full local curation flow, or pick files for read-only curation.

What this local build does

Curation

Stacks, tags, notes, ratings, favorites, focus mode, grid selection, and keyboard sorting stay local.

Metadata

Uses a local JSON sidecar manifest inside the granted root. No EXIF write-back is attempted.

Move

Can move files between subfolders inside the granted root when the browser grants write access.

Delete

Delete moves files into an app-managed trash folder inside the granted root. This is not the OS recycle bin.

If write access is unavailable, curation still works but move/delete actions are disabled.
