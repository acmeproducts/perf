UI V5 – IndexedDB-first Sync Contract

These UI notes mirror performance-v1-sync-plan.txt so the front-end keeps the same invariants as the worker. Every interaction must prefer local state and defer cloud writes to the background worker.

Cold Folder Bootstrap

  • Kick off one provider listing to collect app properties and <fileId>.json blobs.
  • Write all received metadata into IndexedDB before rendering thumbnails or notes.
  • Stream the PNG range response (or cached bytes) through PNGMetadataExtractor and keep the parsed text-only payload exclusively in IndexedDB.
  • Persist cloud_last_updated_date_time, local_last_updated_date_time, and the folder name as soon as the bootstrap completes.
  • Create an empty, timestamp-stamped queue entry for future add/change/delete operations.

Foreground Editing

  • All sliders, note editors, and stack pickers write directly to IndexedDB; the UI never sends network requests.
  • Append the corresponding queue entry (add, change, or delete) and trigger the debounced inline-blob worker wake-up.
  • Keep the worker off the critical path: it batches queue items, pushes compact payloads to app data and file-id.json, then rechecks the queue before sleeping.
  • Flush the queue when the tab unloads or the folder view closes so no local edits linger.

Device Scenarios

  1. First open on any device: run the full bootstrap once, persist timestamps, and leave the worker idle until a genuine change arrives.
  2. Reopen after edits on the same device: hydrate the view from IndexedDB, apply queued deltas, and update the local timestamp without any provider calls.
  3. First open on a second device: perform a single download, copy the pending-change list, and reuse that data for subsequent renders.
  4. Later opens on that device: compare cloud_last_updated_date_time against the last drained change. If current, render immediately; otherwise fetch only the missing change payloads and merge them.

Guardrails

  • No extra provider traffic: the UI must not emit redundant fetches; background sync is the only network actor.
  • No fingerprinting: never hash file lists or metadata to guess freshness.
  • LRU-bounded IndexedDB: trim least-recently-used folder caches and queues so storage does not expand forever.
  • PNG metadata cache key: always reference the { fileId, pngMetaVersion } composite key before fetching new bytes.
